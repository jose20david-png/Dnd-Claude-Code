◈ RURIK CAMPAIGN SYSTEM

Interactive Architecture & Data Flow Diagram

System Overview

campaign.exe

Node.js bundled executable running three HTTP servers. Loads state from disk, communicates with Anthropic API, manages the DM agentic loop.

Port 3140 — State API

Simple CRUD endpoint for campaign_state.json. Dashboard queries this for inventory, quest status, NPC data.

Port 3141 — DM Relay

Main agentic loop. Receives chat prompt, streams responses via SSE (Server-Sent Events), manages tool execution and history persistence.

Port 3000+ — Dashboard

Single-page web app (HTML + JS). Displays game state, chat, character sheet, inventory. Communicates with ports 3140 and 3141.

GitHub Sync

After every tool call, state is auto-saved. After end_session tool, context file and journal are updated, then git commit + push.

Files on Disk

campaign_state.json (live state), claude_chat_history.json (messages), Campaign Context full.md (session notes), journal.md (recaps).

Agentic Loop (Max 6 Iterations)

Loop Outcomes

✓ End Turn (stop_reason='end_turn')

Claude wrote text and no tools. Break loop, collect finalText, save to history.

⚙️ Tools Called (stop_reason='tool_use')

Execute all tools, add results to messages, continue to iteration N+1 (max 6 total).

✗ API Error (HTTP 400/429/500)

Break loop, set apiError flag, show error message to user, save to history.

⚠️ Tools But No Text (fallback)

If loop produces no final text: trigger 1 emergency follow-up call to prompt narration.

Complete Message Flow

Step 1: User sends message

Browser → Server

POST /api/chat with {prompt: "I cast Guiding Bolt"}

Dashboard disables textarea to prevent interruption.

Step 2: Server loads context

Load → Prepare

Load claude_chat_history.json, add user message, load campaign_state.json, build system prompt with current game state.

Step 3: Agentic loop

Iteration 0

Call Anthropic. Claude responds: text="", tools=[roll_dice, use_spell_slot].

Execute Tools

Roll d20+5 → 18. Use spell slot Lv1. Update state. Send SSE: {type:'state_update', state:{...}}

Iteration 1

Send tool results + nudge. Claude responds: text="The bolt streaks...", tools=[].

Break

stop_reason='end_turn', exit loop with finalText="The bolt streaks..."

Step 4: Stream to browser

SSE Events (ordered)

1. {type:'text', content:"The bolt streaks"} (live narration)
2. {type:'state_update', state:{...}} (HP bar updates)
3. {type:'done', token_count:1234} (re-enable textarea)

Step 5: Persist

Save to Disk

Add assistant message to claude_chat_history.json, save to disk, auto-commit to git.

Updates Trigger State SSE

When a tool modifies state, the server automatically sends {type:'state_update', state:{...}} to the dashboard, which updates all UI elements.

Auto-Saved to Disk

After every state change, campaign_state.json is written to disk and auto-committed to git with a timestamp.

Persists Across Restarts

On startup, launcher.js loads the saved state from disk, so the game continues exactly where it left off.

Campaign Context Syncs

After end_session, the state is used to update Campaign Context full.md with current HP, spell slots, location, and recent events.

Error Handling & Recovery

Scenario 1: HTTP 429 (Rate Limited)

What happens: Anthropic API returns rate limit error.

Server: Detects HTTP 429, extracts error message, rejects with detailed reason.

Console: ✗ Loop 0 API call failed: HTTP 429 — rate_limit_error: ...

Dashboard: Shows ⚠️ DM connection issue: HTTP 429 — rate_limit_error: Requests exceeded

User Action: Wait 10–20 seconds, send message again.

Scenario 2: Tools But No Narration

What happens: Claude calls tools (roll_dice) but writes no text.

Server: After loop, finalText is empty. Trigger fallback: "Please narrate what happens now."

Fallback Result: Still empty (Claude is stuck in tool-only mode).

Dashboard: Shows ⚠️ No DM narration this turn

User Action: Rephrase message or send "continue" to prompt a response.

Scenario 3: Network Timeout

What happens: Browser loses connection during streaming.

Browser: fetch() error caught, shows alert: "Failed to send message. Is the server running?"

User Action: Ensure campaign.exe is running on localhost:3141, then retry.
